## Ключевые этапы и что было сделано
### EDA
- Построен дэшборд при помощи ydata
- Установлено, есть ли в данных пропуски и дубликаты
- Данные в некоторых колонках приведены к числовому формату

- Найдены неполные дубликаты (без таргета)
- рассмотрена статистика по вещественным и категориальными признакам
### Визуализации
- Визуализация попарных числовых признаков для train и test
- Построена тепловая карта корреляций для числовых признаков на train
### Обучение моделей
- Обучена линейная регрессия, метрики R2 test: 0.5941441219017086 MSE test: 233297548204.61337
- Прошлый шаг проделан со стандартизированными данными, результат практически тот же
- Обучена модель Lasso на нормализованных данных, результат почти тот же
- При помощи GridSearch перебраны параметры alpha для Lasso, метрики почти без изменений
- При помощи GridSearch перебраны параметры alpha и l1_ratio для ElasticNet, метрики R2 test: 0.5662248384914819 MSE train: 249346349561.77444
- Далее кодируются категориальные фичи
- На этих данных При помощи GridSearch обучена Ridge и перебраны параметры alpha, метрики R2 train: 0.6537877907498775 MSE train: 99237692378.66805
- Произведена нормализация данных и по аналогии с прошлым пунктом обучена Ridge, удалось незначительно улучшить r2
- Произведена оценка моделей согласно бизнес-метрике, лучший результат показала ElasticNet - 0,237 - она используется в сервисе
### Буст качества
- Наибольший буст качества дало добавление категориальных фич в данные
### Выводы по моделям
- В целом можно заметить, что регрессия дает хороший результат только на данных с нормальным распределением таргета
- К сожалению не удалось получить хороший показатель r2 (хотя бы выше 0,75)
